# PixelRats 指令書 — 第一章：MVP 產品核心定義

> 版本：v0.2（2026/03 — 融合團隊 Feedback 後大改版）
> 狀態：待 Review 鎖定
> 核心改動：等級系統 → Mood + Trust 情感系統；時長 4 種 → 3 種；配件 5 部位 → MVP 3 部位

## 1. 產品定位與核心差異化

**一句話定位：** Focus Tamagotchi — 一隻像素老鼠陪你專注，牠的心情和信任取決於你。

**產品本質：** Tamagotchi + Forest App + NFT Economy。市場上 Forest 只有樹沒有角色，PixelRats 把生產力工具變成情感陪伴。

**使用者每天打開 App 的理由不是 NFT、不是 gacha，而是：「我今天的老鼠過得怎樣？」**

**與 Forest 的核心差異：**
- Forest 是「被動成長的樹」，PixelRats 是「有情感反應的活體陪伴」
- 老鼠有心情（Mood）和信任（Trust）——牠會開心、無聊、難過，而這一切取決於你有沒有陪牠
- 視覺風格：溫暖質感像素風 + 關燈色調（Dark Mode）
- 環境氛圍導向：核心體驗是「打造你跟老鼠一起待的空間」

**目標用戶（優先序）：**
1. 大學生/研究生（考試、寫報告）— 最大量體，社交傳播力強
2. 上班族（深度工作、會議間隙）— 付費意願高
3. 寵物鼠飼主社群 — 量體小但情感共鳴極強，冷啟動傳播核心

**優先序原則：**
```
1. Focus Habit App（專注習慣工具）
2. Rat Bonding（老鼠情感連結）
3. Gacha（抽取收集樂趣）
4. NFT（Web3 經濟層）
```

## 2. 核心循環（Core Loop）

### 情感驅動循環（取代數值驅動）

```
Focus Session（專注）
    ↓
Rat Mood 上升（老鼠變開心）
    ↓
Rat Trust 累積（老鼠更信任你）
    ↓
解鎖新東西（新 gacha 池、新互動、新場景）
    ↓
老鼠變得更可愛
    ↓
玩家繼續 Focus
```

**關鍵設計原則：** 所有系統都只掛在兩個核心變數上——Mood（短期情緒）和 Trust（長期關係）。不做多維度屬性（不做飢餓值、清潔度、社交值），保持極簡。

### 一個完整的專注體驗（四階段）

#### 階段 A：啟動專注（~5 秒）
- 用戶看到老鼠在窩裡的 idle 狀態（動畫反映當前 Mood）
- 選擇專注模式：

| 模式 | 時長 | Mood 獎勵 | Trust 獎勵 | Cheese 獎勵 | 定位 |
|------|------|-----------|-----------|-------------|------|
| Short Focus | 25 分鐘 | +5 | +3 | +10 | 日常輕度專注 |
| Deep Focus | 50 分鐘 | +10 | +5 | +25 | 認真工作/讀書 |
| Flow Focus | 100 分鐘 | +20 | +10 | +60 | 深度心流狀態 |

- 點擊「開始專注」→ 老鼠拿起小書本，坐到書桌旁

#### 階段 B：專注中（25/50/100 分鐘）
- **畫面：** 倒數計時器 ＋ 老鼠在畫面上陪伴念書的動畫
- **老鼠行為：** 根據當前 Mood 展示不同動畫——Mood 高時翻書快、偶爾開心抬頭；Mood 低時打瞌睡、動作慢
- **白噪音：** 可開啟背景音效（與場景綁定）
- **防分心機制（三層式 Grace Period）：**

| 離開時間 | 老鼠反應 | 懲罰 | 設計意圖 |
|---------|---------|------|---------|
| 0–15 秒 | 無變化，繼續念書 | 無 | Grace Period：查字典、看計算機 |
| 15–60 秒 | 抬頭困惑，停止念書 | Mood 不扣，但標記「品質降級」 | 視覺提醒：該回來了 |
| 60 秒以上 | 難過表情，縮起來 | Mood 每 30 秒 -2，Cheese 獎勵打折 | 正式懲罰：確實在分心 |

  - 回來後老鼠需要 3–5 秒「恢復」動畫（困惑→回到念書）
  - 每次 session 的離開事件全部記錄，影響最終結算

#### 階段 C：專注結束 — 獎勵時刻（~15–30 秒）
- **餵食動畫：** 飼料掉落 → 老鼠衝過去吃 → 滿足表情
- **結算面板：** 顯示本次獲得的 Mood、Trust、Cheese
- **Gacha 機會：** 如果 Cheese 累積達到門檻，可以選擇抽取配件
- **Mood 連鎖效果：** 如果 Mood 在 80+ 時完成專注，Cheese 獎勵額外 +20%
- **稀有老鼠預覽：** 低機率觸發稀有老鼠出現（Phase 1 展示用，Phase 2 NFT）

#### 階段 D：休息期（5–15 分鐘）
- 老鼠在窩裡消化休息
- 用戶可以：摸老鼠（+Mood）、查看狀態面板、瀏覽商店、調整場景佈置
- 倒數結束提示下一個專注開始

## 3. Mood + Trust 情感系統（取代等級系統）

### 3.1 Mood（心情）— 短期變數

**範圍：** 0 – 100

| Mood 區間 | 狀態名稱 | 老鼠表現 | 系統影響 |
|-----------|---------|---------|---------|
| 0–20 | Sad（難過）| 縮在角落、不太動 | Gacha luck -20%、Trust 增長減半 |
| 20–40 | Bored（無聊）| 發呆、偶爾嘆氣 | 無加成無懲罰 |
| 40–60 | Normal（普通）| 正常活動 | 基礎狀態 |
| 60–80 | Happy（開心）| 活潑亂跑、搖尾巴 | Gacha luck +10%、Trust 增長 +25% |
| 80–100 | Excited（興奮）| 跳來跳去、發光效果 | Gacha luck +20%、Cheese 獎勵 +20% |

**Mood 變化來源：**
- 完成專注 session → +5/+10/+20（依時長）
- 摸老鼠（休息期互動）→ +3（每日上限 3 次）
- 每日登入 → +5
- 離開 App（專注中，超過 60 秒）→ 每 30 秒 -2
- 自然衰減 → 每小時 -1（離線時緩慢下降）
- 連續未開 App → 每天 -5

**Mood 底線規則（防棄遊設計）：**
- **離線衰減底線：30（Bored）**——不管多久沒開 App，老鼠最差就是「無聊」，不會「難過」
- **Sad 狀態（0–20）只能由專注期間離開 App 觸發**——這是玩家主動行為的後果，不是離線懲罰
- **設計哲學：** 回歸用戶看到的是「無聊等你回來的老鼠」，而不是「痛苦怪你拋棄的老鼠」

**設計意圖：** Mood 讓玩家有「今天要讓老鼠開心」的短期目標。開心的老鼠抽卡更幸運，這創造了「先專注 → 老鼠開心 → 再抽卡」的行為循環。

### 3.2 Trust（信任）— 長期變數

**範圍：** 0 – 1000

| Trust 區間 | 關係階段 | 解鎖內容 |
|-----------|---------|---------|
| 0 | Stranger（陌生人）| 基礎功能 |
| 100 | Friend（朋友）| 解鎖 Basic Gacha Pool |
| 300 | Partner（夥伴）| 解鎖 Rare Gacha Pool |
| 600 | Family（家人）| 解鎖 Epic Gacha Pool |
| 1000 | Soul Bond（靈魂羈絆）| 解鎖 Legendary Gacha Pool + 特殊稱號 |

**Trust 變化來源：**
- 完成專注 session → +3/+5/+10（依時長）
- Mood 60+ 時完成專注 → Trust 額外 +25%
- 每日登入（streak）→ +2（連續 7 天以上 +5）
- Trust **不會衰減**（長期累積，獎勵忠誠用戶）

**設計意圖：** Trust 是長期留存的核心。玩家花了幾個月累積到 600 Trust 解鎖 Epic Pool，不會輕易放棄這個 App。Trust 門檻控制 gacha pool，讓新玩家不會一開始就能抽到最好的東西。

### 3.3 Mood 和 Trust 的關係

```
Mood = 短期動力（今天有沒有陪老鼠）
Trust = 長期承諾（這幾個月有沒有持續陪老鼠）

Mood 影響 Trust 的增長速度：
  Mood 高 → Trust 漲更快
  Mood 低 → Trust 漲更慢（但不會倒扣）

Trust 影響 Mood 的底線：
  Trust 高 → Mood 自然衰減更慢（老鼠更信任你，不會那麼快不開心）
  Trust 低 → Mood 衰減正常速度
```

## 5. Gacha 抽取系統 — Phase 1 實裝

> **戰略定位：** Gacha 是 Mood + Trust 系統的「獎勵輸出口」。Mood 影響抽卡運氣，Trust 決定能進入哪個抽卡池。配件保留完整 metadata，作為未來育鼠系統的基因庫。

### 5.1 Gacha 貨幣

只有兩種，保持極簡：

| 貨幣 | 取得方式 | 用途 |
|------|---------|------|
| 🧀 Cheese（起司）| 專注獲得、每日登入、活動 | 免費抽取 |
| 🧀 Gold Cheese（黃金起司）| IAP 購買 | 付費抽取（機率更高） |

### 5.2 配件部位（MVP 3 部位）

| 部位 | slot 值 | 說明 | 階段 |
|------|---------|------|------|
| 頭飾 | head | 帽子、髮飾、頭巾 | ✅ MVP |
| 背部裝飾 | back | 翅膀、披風、背包 | ✅ MVP |
| 背景 | background | 老鼠專屬背景（僅預覽/截圖顯示） | ✅ MVP |
| 嘴飾 | mouth | 口罩、鬍子、食物 | ❌ Phase 1.5 |
| 毛色 | pattern | 基底色系替換 | ❌ Phase 1.5 |

### 5.3 稀有度與機率

| 等級 | 顏色標識 | 基礎機率 | Mood 80+ 時機率 | 視覺特效 |
|------|----------|----------|----------------|----------|
| Legend | 金色 | 5% | 6% | 發光＋粒子效果 |
| Epic | 紫色 | 10% | 12% | 微光效果 |
| Rare | 藍色 | 25% | 27% | 輕微光暈 |
| Common | 灰色 | 60% | 55% | 無特效 |

> 註：原本的 SS/S/A/B/C 五級簡化為 Legend/Epic/Rare/Common 四級，降低認知負擔，與 Trust 門檻的四個 gacha pool 完美對應。

### 5.4 Trust 門檻 × Gacha Pool

| Trust 等級 | 可進入的 Pool | Pool 內容 |
|-----------|-------------|-----------|
| 0+ Stranger | Basic Pool | 僅 Common 配件 |
| 100+ Friend | Basic + Rare Pool | Common + Rare 配件 |
| 300+ Partner | 上述 + Epic Pool | 解鎖 Epic 配件 |
| 600+ Family | 上述 + Legend Pool | 解鎖 Legend 配件 |

**設計意圖：** 新玩家抽不到好東西，不是因為運氣差，而是因為「老鼠還不信任你」。這把冰冷的機率門檻包裝成了情感敘事。

### 5.5 配件總量規劃（MVP）

- **MVP 目標：** 3 部位 × 4 級 × 3–4 款/級 = **約 36–48 件配件**
- **比原方案減少 50%+**，大幅降低美術工作量
- 美術產製方式：PixelLab.ai → Aseprite 精修 → sprite sheet

### 5.6 抽取規則

- **貨幣：** Cheese（免費）或 Gold Cheese（付費）
- **抽取成本：** 待第三輪定案
- **重複處理：** 可抽到重複配件，重複品可分解回收為 Cheese
- **Mood 影響：** Mood 80+ 時 luck +20%（高稀有度機率提升）；Mood < 20 時 luck -20%
- **保底機制：** 待第三輪定案

### 5.7 Metadata 結構（為育鼠系統繼承預留）

每件配件保留完整 metadata：
- `item_id`：全域唯一 ID
- `slot`：部位（head / back / background / mouth / pattern）
- `rarity`：稀有度（legend / epic / rare / common）
- `name`：配件名稱（中/英雙語）
- `sprite_ref`：對應 sprite sheet 路徑
- `obtained_at`：獲取時間戳
- `obtained_by`：擁有者 user_id
- `source`：獲取方式（gacha / event / reward）
- `is_equipped`：是否裝備中
- `gene_data`：育鼠繼承用（inheritable, gene_weight）

## 6. NFT 稀有老鼠與育鼠系統（Phase 2 實裝，Phase 1 架構預留）

> **核心概念：** NFT 是一隻完整的「稀有老鼠夥伴」。牠會無時無刻陪伴玩家的主角老鼠，並可透過育鼠系統繁殖 Gen2。

### 6.1 NFT 稀有老鼠（Gen1）
- **取得方式：** App 內購買抽取（信用卡 / Apple Pay），零錢包門檻
- **功能：** 裝備後，NFT 老鼠會在所有場景中陪伴主角老鼠互動
- **獨立外觀：** 毛色基底、體型、尾巴樣式、眼睛顏色等

### 6.2 育鼠系統（Breeding）
- **配對範圍：** Gen1 NFT × 主角老鼠，或 Gen1 NFT × 其他玩家的 Gen1 NFT（社交育鼠）
- **繁殖次數：** Gen1 每隻終生只能繁殖 **1 次**
- **Gen2 不可再繁殖：** 世代限定兩代

### 6.3 Gen2 NFT 繼承邏輯
```
Gen1 NFT 老鼠的 items ─┐
                        ├→ 隨機抽取繼承 ＋ 小機率突變出全新 items → Gen2 NFT
玩家主角老鼠的 items ──┘
```
- Gen2 從雙親 items 池中隨機抽取部分 items
- 小機率突變出全新 items（突變限定款）
- 高稀有度 items 有較高繼承權重（具體數值待第三輪定案）

### 6.4 NFT Metadata 結構
- `nft_id`：鏈上唯一 token ID
- `generation`：世代（gen1 / gen2）
- `parent_ids`：雙親 ID
- `base_appearance`：基底外觀
- `inherited_items[]`：繼承的配件列表
- `mutated_items[]`：突變產生的新配件列表
- `breed_count` / `breed_available`：繁殖狀態
- `minted_at` / `owner_id`

## 7. 留存機制（MVP 版本）

### 7.1 Streaks 系統（融入 Mood + Trust）
- **每日登入：** Mood +5、Trust +2、送免費 Cheese
- **連續天數加成：**
  - 連續 3 天 → Cheese ×1.5
  - 連續 7 天 → Trust +5（取代 +2）、解鎖特殊家具
  - 連續 30 天 → 待定獎勵
- **斷連處理：** 待第三輪討論定案

### 7.2 Mood 自然衰減（溫和回召機制）
- 離線時 Mood 每小時 -1
- 連續未開 App 每天額外 -5
- **離線衰減底線：Mood 30（Bored）**——絕不會因離線掉到 Sad
- Sad（0–20）只能在專注期間離開 App 時觸發
- 玩家回來時看到的是「無聊等待的老鼠」→ 溫和召喚而非內疚懲罰

### 7.3 Trust 不衰減（獎勵長期忠誠）
- Trust 只增不減，永不倒扣
- 長期用戶即使中斷一段時間回來，Trust 等級還在
- 這降低了「我中斷太久所以乾脆不回來」的棄遊心理

## 8. 社交功能（MVP 版本）

- **一鍵截圖分享：** 自動生成精美截圖（老鼠 ＋ 配件背景 ＋ 今日 Mood/Trust 數據）
- **分享管道：** IG Story、LINE、X（Twitter）
- **不做：** 好友系統、排行榜、即時互動（Phase 1.5+）

---

## 9. 語言支援

- **MVP 雙語：** 繁體中文（主要）＋ 英文
- **架構需求：** 從第一天就用 i18n 架構（flutter_localizations + ARB）
- **UI 文案風格：** 溫暖、輕鬆、帶一點幽默（老鼠的對話泡泡語氣根據 Mood 狀態變化）

---

## 10. 老鼠設定

- **數量：** 一隻主角老鼠（用戶命名）
- **主角老鼠的變化：** 根據 Mood 和 Trust 等級展示不同動畫和行為
- **稀有老鼠預覽（Phase 1）：** 結算時低機率出現，讓玩家認識稀有老鼠存在，為 Phase 2 NFT 預熱
- **Personality 人格系統：** Phase 2 才做（Lazy/Genius/Greedy 等人格作為 stat modifier）
